    Create src/sfgraph/parser/worker/worker.js using the RESEARCH.md Pattern 1 as the canonical template. Key implementation rules:

    1. `const { getApexParser } = require('web-tree-sitter-sfapex');` — only this package, no native tree-sitter
    2. Grammar loaded ONCE in `init()` async function — NOT inside handleLine
    3. `readline.createInterface({ input: process.stdin, terminal: false })` on stdin
    4. handleLine() dispatches: type===ping → pong; type===exit → process.exit(0); otherwise → parse
    5. APEX-10 guard: `if (root.hasError)` — property access, NO parentheses
    6. fileCount incremented BEFORE checking MAX_FILES (200). When fileCount > MAX_FILES on a parse request: return {ok:false, error:"memory_ceiling"} then call process.exit(0)
    7. Successful parse returns: {requestId, ok:true, payload:{filePath, hasError:false, nodeCount, nodes:[], potential_refs:[]}}
    8. All logging goes to process.stderr.write() — NEVER process.stdout.write() for logs
    9. Wrap entire handleLine in try/catch; exception → {ok:false, error:e.message, payload:null}
    10. init() failure → process.stderr.write + process.exit(1)
