<interfaces>
<!-- worker.js IPC protocol (from Plan 01) -->
Parse request (Python → Node.js):
  {"requestId": "<uuid4>", "grammar": "apex", "filePath": "<path>", "fileContent": "<source>"}

Parse response (Node.js → Python):
  ok:true  → {"requestId": "<uuid4>", "ok": true, "payload": {"filePath": ..., "nodeCount": N, "nodes": [], "potential_refs": []}}
  ok:false → {"requestId": "<uuid4>", "ok": false, "error": "<reason>", "payload": null}
  Special  → {"requestId": "<uuid4>", "ok": false, "error": "memory_ceiling"} + worker self-exits

Ping/pong:
  Send: {"requestId": "<uuid4>", "type": "ping"}
  Recv: {"requestId": "<uuid4>", "type": "pong"}

Exit signal: {"type": "exit"}

<!-- Node binary resolution (from research): -->
NODE_BINARY = "/opt/homebrew/opt/node@22/bin/node"
WORKER_JS = Path(__file__).parent / "worker" / "worker.js"
Fallback: shutil.which("node")

<!-- POOL-06 critical constraint from research pitfall #4: -->
After asyncio.TimeoutError, the worker is still alive and its response is buffered in its stdout pipe.
Resolution: each request has a unique requestId; after timeout, the worker's pending response
will be read and discarded by correlating requestId on the NEXT call, OR simply mark the
worker as needing a drain. Simplest safe approach: after timeout, replace the worker
(treat same as failed health check) — this avoids stale response contamination entirely.
Per-file timeout (POOL-06) returns error WITHOUT killing the worker, but the worker
is then treated as suspect and replaced via _replace_worker().
</interfaces>

  <action>
    Create src/sfgraph/parser/pool.py following the RESEARCH.md Pattern 2 exactly. Implementation notes:

    1. Module-level constants:
       - NODE_BINARY = "/opt/homebrew/opt/node@22/bin/node"
       - WORKER_JS = str(Path(__file__).parent / "worker" / "worker.js")

    2. _resolve_node() helper: checks if NODE_BINARY path exists; falls back to shutil.which("node"); raises RuntimeError if neither found.

    3. _Worker dataclass: proc (asyncio.subprocess.Process), semaphore (asyncio.Semaphore(1)), healthy (bool=True).

    4. NodeParserPool.__init__(size=None): self._size = size or min(os.cpu_count() or 4, 8); self._workers = []; self._health_task = None; self._shutdown = False.

    5. start(): spawn self._size workers via _spawn_worker(); then create_task(_health_loop()). Start health loop AFTER all workers are ready.

    6. _spawn_worker(): asyncio.create_subprocess_exec(_resolve_node(), WORKER_JS, stdin=PIPE, stdout=PIPE, stderr=PIPE). Returns _Worker.

    7. parse(file_path, grammar, file_content): find first healthy worker where semaphore is not locked. If all busy, use self._workers[0]. Acquire semaphore, call _dispatch().

    8. _dispatch(worker, file_path, grammar, content):
       - Generate uuid4 requestId
       - Write JSON + "\n" to worker.proc.stdin
       - asyncio.wait_for(worker.proc.stdout.readline(), timeout=10.0)
       - On TimeoutError: replace worker via asyncio.create_task(_replace_worker(worker)); return {"ok":False,"error":"timeout","payload":None}
       - On memory_ceiling response: schedule _replace_worker; return {"ok":False,"error":"worker_restarting","payload":None}
       - Otherwise: return parsed JSON result

    9. _health_loop(): while not self._shutdown: await asyncio.sleep(30); for each worker, if not await _ping_worker(worker): schedule _replace_worker.

    10. _ping_worker(worker): send {"requestId":uuid, "type":"ping"}, wait_for readline timeout=5.0; return result.get("type")=="pong"; catch all exceptions → return False.

    11. _replace_worker(old_worker): set old_worker.healthy=False; SIGKILL + await; spawn new worker; replace in self._workers list at original index.

    12. shutdown(): self._shutdown=True; cancel health_task; kill+wait all workers.
  </action>
